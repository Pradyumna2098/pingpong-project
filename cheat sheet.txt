Clean Code Development

1)Use proper explanatory names for defining variables.
2)Use empty lines between the code for better readability.
3)Use unit tests and practice Test Driven Development.
4)Create descriptive messages while committing.
5)Good commenting on important lines in the code grabs the interest of others viewing the code.
6)Error handling -> throw proper exemptions
7)Functions should do one thing and be named uniquely.
8)Do not add more than 3 parameters in a function.
9)Use parameter and return types in functions and classes.
10)Code refactoring whenever needed.
11)Note the version control of what is being used in the project.
12) Introduce your own CI/CD pipelines for the project.
